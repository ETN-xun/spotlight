一．	核心概述
1.1	游戏类型：回合制策略游戏
1.2	比赛题目：Bug？你确定这不是个Bug吗？
1.3 目标平台：PC
1.4 核心玩法概述：
这是一款回合制战棋策略游戏，玩家通过控制己方棋子，在限定的行动数中完成具体任务。
1.5 DEMO目标：
1.5.1 制作三种我方单位（棋子）和3-5种敌方单位（棋子），3-5种地形
1.5.2 验证核心机制
1.5.3 构建三个可游玩关卡
1.6 美术风格
故障艺术风格
像素撕裂、色彩溢出、信号干扰、扫描线抖动
1.7 背景故事
虚拟世界架空背景
剧情梗概：未来时间，一个元宇宙虚拟世界产生崩坏，bug横生，主角临危受命，和小队一起击败虚拟敌人，铲除一个一个bug，恢复虚拟世界秩序，寻找失序的真相
二．	核心机制
2.1 玩家控制：鼠标控制
2.2 定位：基于方形网格的回合制战棋
2.3 游戏地图：随机生成的8*8方格棋盘地形
2.4 机制介绍
2.4.1 完美信息
所有敌方单位下回合的攻击目标、伤害值和行动顺序完全可见。消除了传统战棋的“未命中焦虑”和随机性挫败感，将挑战从“概率计算”完全转变为 “谜题求解”
2.4.2 行动经济
在每个我方回合中，我方单位可以进行移动或使用技能，这二者都将消耗行动能量。我们将设置一个整体能量池（系统带宽），每个我方单位都可以使用若干能量行动，能量耗尽后不能继续行动。
   能量回复：每回合我方基础回复6点能量
系统带宽设定：
占领缓存区可以额外回复2点能量，能量最多不超过10点
击杀空指针或死机亡灵（特定敌人），额外回复能量
2.4.4 每局游戏目标
击败所有敌人
保护特定资源
击败boss
2.5 战斗模式
部署阶段：在8*8方格地图的一定范围内部署我方单位
敌人行动回合：每个敌人可以进行一次位移并瞄准攻击目标，部分区块产生预备敌人（下回合开始行动）
我方行动回合：
合理分配能量点数进行位移和攻击
可以查看敌人下回合行动顺序，攻击位置；也可以点击查看敌方单位，我方单位，地形的详细介绍
敌方行动回合：
先对上一回合的目标方块进行攻击，之后再次位移并锁定新的单位
2.6 敌我单位设计
2.6.1 我方单位
我方将会设计三个单位
所有单位外观采用故障艺术风格，身体边缘有像素撕裂、色彩分离特效
1.	代码刺客-影：移动4格，攻击一格，近战输出
能力：
断点斩杀：能量消耗1，技能范围1，基础伤害2，若敌人处于状态异常状态，直接秒杀；攻击时会将目标推动一格，推动若撞到其他单位或地形、建筑，此单位和撞击单位均受一点伤害
闪回位移：能量消耗1，无技能范围和伤害，效果为在此位置留下一个嘲讽敌人的残影，同时本体回到上次移动前的位置
2.	协议卫士-石：移动两格，攻击3格，坦克控制
能力：
堆栈护盾：能量消耗1，技能范围4，无伤害，可以为友军，地形，建筑提供一个能吸收一次伤害的护盾，护盾受到伤害碎裂，否则一直存在
地形投放：能量消耗1，技能范围4，技能伤害1，远程投掷一个防火墙，若砸中敌人，地形，或建筑，造成一点伤害，同时投掷物损毁，若投掷至空地形，则形成一个可以抵挡一次攻击的墙，受击损毁，不受击则一直存在
3.	指针操控者-零：移动三格，远程攻击，无视地形
能力：
强制迁移：能量消耗1，技能范围8，技能伤害1，将目标向自身1拉动一格，若拉动时撞到其他单位或地形、建筑，此单位和撞击单位均受一点伤害
移形换影：能量消耗1，技能范围1，无伤害，将面前的敌/我单位与场上任意位置的敌/我单位交换位置
2.6.2 敌方单位
将会设计3-4种敌人
1.死机亡灵：高攻高防，有概率本回合不行动（死机）
2.乱码爬虫：近战，低血量
死亡感染：死亡时对周围造成一点腐蚀伤害，对伤害目标造成状态异常效果
3.空指针：刺客
闪现：可任意位移到一格，但无法连续攻击同一目标两回合
5.递归幻影(Boss):复制，召唤
镜像复制：复制一个我方单位
递归召唤：每三回合召唤一个乱码爬虫
2.6.3 特殊技能
过热降频：如果一个单位行动次数（移动次数+使用技能次数）>3，则在下一回合为其添加 （移动力-1，技能伤害-1，作为可调整参数）。
过载模式：全局技能，在GameManager中实现。我方回合时选择消耗4能量，遍历所有我方单位，清除其所有负面状态。并让我方所有单位进入过载模式，所有加成效果只持续我方行动回合一回合
代码刺客：（无限火力，攻击不消耗能量点）本回合移动力+2，伤害+1，攻击后可不消耗能量移动两格，若消灭目标，则本回合下次攻击不消耗能量点
协议卫士：为两个友军或建筑和自己添加护盾，伤害+1，范围+1
指针操控者：使用技能互换位置时，若其中包含敌方单位，控制其下回合无法行动并施加状态异常，本回合技能伤害+1。
状态异常效果：挂载此效果的单位受到的伤害+1
三．地形与特殊地块
Bug格子:停留在格子上的单位生命值和攻击力数值调换，施加状态异常效果
数据平原：无特殊效果，有轻微像素抖动
腐蚀区块：对停留单位造成一点伤害并施加状态异常效果
防火墙：不可通行，受到两次攻击后消失
缓存区：占领后每回合额外回复2点能量，受到一次攻击后损毁
四.剧情与背景
4.1 世界观设定
时间：2077年
地点：元界系统——一个由人类意识与数据流共同构建的虚拟现实世界。
元界系统原是人类社会的延伸，承载着记忆、文化、社交、经济甚至政治活动。系统由“核心协议”维持运转，而“调试者”是一群拥有高级权限的虚拟守护者，负责维护系统稳定。
然而，一场名为“腐码风暴”的灾难性事件爆发，导致系统逻辑层崩坏，现实与虚拟的边界模糊，Bug实体化为具有攻击性的“腐化体”，不断吞噬数据流、扭曲空间结构，威胁整个元界——乃至现实世界的稳定。
4.2 背景故事
前因：
腐码风暴的起源，源于一次未被授权的“递归意识上传”实验。却  在过程中触发了未被记录的底层协议漏洞，导致“腐码”如病毒般扩散。腐码不仅扭曲了系统规则，更唤醒了系统中沉睡的“遗留代码”——那些被遗忘的程序碎片、错误指令、未关闭的进程——它们化为实体，成为“腐化体”。
后果：
系统陷入混乱，区域崩溃、数据丢失、虚拟居民“意识迷失”。调试者组织派出多支小队前往腐化区执行清理任务，但大多失联。唯一传回的信息是一段被干扰的讯号：“腐码……有意识……”

4.3 剧情演绎方式（借鉴《漫野奇谭》风格）
•	分镜漫画式叙事：关键剧情以多格漫画呈现，画面中加入像素撕裂、色彩溢出、信号干扰、扫描线抖动等故障艺术效果，增强沉浸感与系统崩坏的视觉表现。
•	动态对话系统：角色对话时，文字会随系统稳定性变化。在腐化区域，对话可能出现乱码、重复、跳字、倒序，甚至被“病毒文本”覆盖，玩家需点击“清理”按钮才能恢复原文。

核心属性与字段		
属性/字段名	类型	说明
CurrentEnergy	int	当前能量值。这是整个系统最核心的变量。
MaxEnergy	int	能量上限，初始为10。
BaseEnergyPerTurn	int	每回合基础回复量，初始为6。

关键方法			
方法名	参数	返回类型	说明
Initialize	void	void	游戏开始时调用，设置初始能量（如开局为6）。
RefreshTurnEnergy	void	void	在玩家回合开始时调用。主要职责是：在每个玩家回合开始时，自动为玩家补充能量，并确保能量不超过上限
TrySpendEnergy	int cost	bool	行动消费的入口。检查能量是否足够，如果足够则扣除并返回true，否则返回false。
GainBonusEnergy	int amount	void	用于击杀特定敌人或占领缓存区时的额外能量获取。直接增加 CurrentEnergy，但不超过上限。

关键事件		
事件名	参数	触发时机
OnEnergyChanged	int current, int max	任何时候 CurrentEnergy 发生变化时。UI系统监听此事件以更新显示。

具体敌人AI行为说明			
敌人类型	移动目标选择	攻击目标选择	地形交互行为
乱码爬虫	移动并瞄准距离最近的我方单位或缓存区，避开腐蚀区块	攻击范围内最近的我方单位、缓存区	死亡时尽量在人群中使用"死亡感染"
			
死机亡灵	移动至能攻击到我方单位或缓存区的位置，优先站Bug格子，避开腐蚀区块	优先攻击范围内最近的单位、缓存区	利用Bug格子获得更高攻击
			
空指针	使用"闪现"移动到能攻击到我方单位的最近位置，避开腐蚀区块	优先攻击未被其上一回合攻击的单位或缓存区	 利用闪现跨越防火墙、缓存区
			
递归幻影(Boss)	保持在中距离（攻击范围内），避开腐蚀区块	优先攻击我方单位
